An efficient algorithm to convert sequential GPS data into Grids

Erdem Koç

One of the most common operations before storing network measurement data into databases of any type is to aggregate the information over grids of desired size. This makes it easier to visualize and proccess the KPIs. Not to mention, it also takes up much less storage space.

One can think that geo-hash is one of the most computationally efficient ways of doing this and he would be right, but it has its own draw-backs and it is not uncommon to use geospatial datataypes for such a task.

Below I implement a fast grid algorithm to assign gps coordinates into grids (of geospatial dataype in a database or similar environment).

It must be noted that I used shapely.geometry.polygon to create geo-spatial datatypes. In a practical real-life scenario, polygons would most likely be stored in a database, or belong to a GIS software. In either case, complexity of the algorithm would still be same, although actual processing times may vary.

Two major criteria this algorithm exploits are as follows:

  1. A coordiante follows the next one most of the time in space. (i.e. belong to a route from a drive-test etc.)
  2. We want to assign grids to rectangular grids.

If the coordinate data satisfies the above given charactheristics then we can think as follows:

Here it is very well explained that, if the polygon we want check whether the point is inside or not is a rectangle; then we can use direct calculation of complexity O(1), hence constant time.

Following functions implement that algiorthm.

def triangleArea(Ax, Ay, Bx, By, Cx, Cy):
    return (Cy*Bx-By*Cx)-(Cy*Ax-Ay*Cx)+(By*Ax-Ay*Bx)
def insideSquare(Ax, Ay, Bx, By, Cx, Cy, Dx, Dy, Px, Py):
    if (triangleArea(Ax, Ay, Bx, By, Px, Py)>0) | \ 
            (triangleArea(Bx, By, Cx, Cy, Px, Py)>0) | \ 
                (triangleArea(Cx, Cy, Dx, Dy, Px, Py)>0) | \ 
                     (triangleArea(Dx, Dy, Ax, Ay, Px, Py)>0):
        return False
    else:
        return True

We crate 2 grid spaces for 2 algortihms and plot it for viusalization

  • 1st algorithm (simple) takes every point and check if that point is inside of any of the polygons we created. No exploitation of sequantial coordinated and/or rectangular girds. (grids cloud as well be of any shape)
  • second algorithm (fast) exploits te points mentioned above
def formGridSpace(scale=1, maxm=30, plot=True):
    '''
    forms a scalable grid structure and plots it
 
    scale=scale factor
    maxm=nr of grids in x adn y direction
 
    returns axes and list of polygons each of which is a grid'''
 
    if plot: 
        fig = plt.figure(figsize=(8,8))
        ax = fig.add_subplot(121)    
        ax2 = fig.add_subplot(122)
        ax.set_aspect(1)
        ax2.set_aspect(1)
        ax.set_title("Simple Grid")
        ax2.set_title("Fast Grid")
    else:
        ax=None
        ax2=None
 
 
    grids = list()
 
    for x in range(0,maxm):
        for y in range(0,maxm):
            coord = [ (scale*0+scale*x, scale*0+scale*y)
                    , (scale*0+scale*x, scale*1+scale*y)
                    , (scale*1+scale*x, scale*1+scale*y)
                    , (scale*1+scale*x, scale*0+scale*y)
                    , (scale*0+scale*x, scale*0+scale*y)]
 
            grid = Polygon(coord)       
            grids.append(grid) 
 
            if plot:
                ring=LinearRing(coord)
                x_, y_ = ring.xy
                ax.plot(x_, y_, color="lightblue")
                ax2.plot(x_, y_, color="lightblue")            
 
    return ax, ax2, grids
# form and plot blank grid spaces 
ax, ax2, grids = formGridSpace()

def artificialGpsRoute(density=1, scale=1): 
    ''' an exponential line to mimic a gps route.

def simpleGrid(route, grids):
    '''for each gps coordinate in route earches a grid polygon
    that the coordinate is in'''
 
    found_grids = list()
 
    for point in route:
        for grid in grids:
            if grid.contains(Point(point[0],point[1])):
                found_grids.append(grid)
                break
 
    return found_grids

Now lets implement a function that uses all the points we mentioned above

Note that even if the points are not sequantial in space the following function would work, but it would fail if the grids are not rectangular. 

def fastGrid(route, grids):
    '''for each gps coordiante searches the grid and if found checks if 
    the following coordiantes are still in that grid. Searches only if the 
    following coordiante is outside'''
 
    grid_found = False
    found_grids = list()
 
    for point in route:
        # search the coordinate in all space 
        if not grid_found:       
            for grid in grids:
                if grid.contains(Point(point[0],point[1])):   
                    # when we find the grid we store coordiantes for the  
                    # rest of the coordiantes 
                    grid_found=True
                    coord = grid.exterior.coords.xy
                    found_grids.append(grid)               
                    break
        else:
            # as long as next point is in the same grid do nothing 
            if insideSquare(coord[0][0], coord[1][0], coord[0][1], coord[1][1]
                            ,coord[0][2] ,coord[1][2] ,coord[0][3] ,coord[1][3] ,point[0] ,point[1]):
                pass 
            else:
                grid_found=False  
 
    return found_grids
